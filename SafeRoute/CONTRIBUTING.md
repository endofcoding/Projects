# Contributing to SafeRoute

Thank you for your interest in contributing to SafeRoute! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs
- Check if the bug has already been reported in Issues
- If not, create a new issue with:
  - Clear title and description
  - Steps to reproduce
  - Expected behavior
  - Actual behavior
  - Environment details (OS, device, app version)

### Submitting Features
- Open an issue to discuss the feature first
- Wait for feedback before starting development
- Include use cases and benefits in your proposal

### Pull Requests
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -m 'Add feature: ...'`
4. Push to branch: `git push origin feature/your-feature`
5. Open a pull request with description of changes

## Code Standards

### Format
- Use Prettier for code formatting
- Run `npm run format` before committing

### Linting
- Follow ESLint rules
- Run `npm run lint:fix` to auto-fix issues

### Type Safety
- Use TypeScript for new code
- Add type annotations to functions
- Run `npm run type-check` to verify

### Testing
- Write tests for new features
- Run `npm test` before submitting PR
- Aim for >80% code coverage

## Commit Messages
- Use clear, descriptive messages
- Start with verb: `Add`, `Fix`, `Update`, `Remove`
- Example: `Add route safety score calculation`

## Questions?
- Open an issue labeled `question`
- Contact the team at support@saferoute.com

Thank you for making SafeRoute better!
